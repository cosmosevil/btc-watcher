# 🔔 BTC Wallet Watcher

Бот, который следит за входящими транзакциями на ваш Bitcoin-кошелёк и присылает уведомления прямо в Telegram. Работает бесплатно через GitHub Actions — никакого сервера держать не нужно.

## Что умеет

| Событие | Сообщение в Telegram |
|---|---|
| Новая транзакция попала в мемпул | 🟡 Замечена + сумма в BTC / USD / RUB + примерное время ожидания |
| Транзакция всё ещё не подтверждена | ⏳ Напоминание с обновлённой оценкой времени |
| Транзакция подтверждена в сети | ✅ Подтверждена + количество подтверждений + сумма в BTC / USD / RUB |
| Ничего нового | ℹ️ Новых транзакций нет |
| Каждый день в 23:59 МСК | 📅 Сводка за сутки — количество транзакций, итоговая сумма в BTC / USD / RUB, список каждой транзакции |

Каждое сообщение содержит кликабельную ссылку на транзакцию в [mempool.space](https://mempool.space). Курс BTC подтягивается в реальном времени через [CoinGecko API](https://www.coingecko.com/).

## Установка за 5 шагов

### Шаг 1 — Создайте Telegram-бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Придумайте имя и username для бота
4. Сохраните **токен** вида `123456789:ABC-...` — он понадобится позже

> ⚠️ Никому не передавайте токен и не вставляйте его в открытый текст — это полный доступ к вашему боту.

### Шаг 2 — Узнайте свой Telegram chat_id

1. Найдите в Telegram бота [@userinfobot](https://t.me/userinfobot)
2. Нажмите `/start`
3. Он пришлёт ваш `Id` — это и есть ваш `chat_id`

### Шаг 3 — Сделайте форк репозитория

1. Нажмите кнопку **Fork** в правом верхнем углу этой страницы
2. Выберите свой аккаунт
3. Убедитесь, что репозиторий **публичный** (Public) — для приватных репозиторий GitHub может блокировать scheduled workflows на бесплатном тарифе

### Шаг 4 — Добавьте секреты

В своём форке зайдите в **Settings → Secrets and variables → Actions → New repository secret** и добавьте три секрета:

| Имя | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен от BotFather (шаг 1) |
| `TELEGRAM_CHAT_ID` | Ваш chat_id (шаг 2) |
| `BTC_ADDRESS` | Bitcoin-адрес кошелька, который отслеживаем |

> 🔒 Секреты зашифрованы и никогда не попадают в код репозитория.

### Шаг 5 — Запустите первый раз вручную

1. Откройте вкладку **Actions** в репозитории
2. Слева выберите **Check BTC wallet**
3. Нажмите **Run workflow → Run workflow**
4. Через 20–30 секунд обновите страницу — должен появиться зелёный запуск ✅
5. Зайдите в Telegram — бот должен был прислать первое сообщение

Дальше всё работает само — GitHub будет запускать проверку каждые ~5 минут автоматически.

## Как это работает

```
GitHub Actions (cron каждые ~5 минут)
        ↓
  btc_watcher.py
        ↓
  mempool.space API  →  проверяет ваш BTC-адрес
  CoinGecko API      →  получает курс BTC в USD и RUB
        ↓
  Новое событие?
    ├── Да → отправляет сообщение в Telegram (с суммой в BTC / USD / RUB)
    └── Нет → "Новых транзакций нет."
        ↓
  btc_watcher_state.json  →  коммитится обратно в репозиторий
                              (чтобы не дублировать уведомления)

GitHub Actions (cron каждый день в 23:59 МСК)
        ↓
  btc_watcher.py --daily-summary
        ↓
  Читает историю подтверждённых транзакций за последние 24 часа
        ↓
  Отправляет сводку в Telegram (количество, итого в BTC / USD / RUB, детали)
```

Файл `btc_watcher_state.json` — это память бота: он запоминает, какие транзакции уже видел, про какие уведомлял, и хранит историю подтверждённых транзакций для ежедневной сводки. При каждом запуске Actions этот файл обновляется и сохраняется в репозиторий.

## Часто задаваемые вопросы

**Почему уведомления приходят не ровно каждые 5 минут?**
GitHub Actions не гарантирует точное время запуска по расписанию — в периоды нагрузки задержка может составлять от нескольких минут до получаса. Это особенность платформы, а не баг бота.

**Почему репозиторий должен быть публичным?**
Для приватных репозиторий на бесплатном тарифе GitHub считает минуты Actions против платёжного лимита ($0 по умолчанию), из-за чего scheduled-запуски могут тихо блокироваться. В публичных репозиториях минуты Actions бесплатны без ограничений.

**Безопасно ли хранить адрес кошелька в репозитории?**
Адрес Bitcoin-кошелька — публичная информация, его знание не даёт доступа к средствам. Токен бота и chat_id хранятся в зашифрованных Secrets и в код не попадают.

**Можно ли отслеживать несколько адресов?**
В текущей версии — один адрес. Поддержку нескольких адресов можно добавить при желании.

**Что делать, если уведомления перестали приходить?**
Зайдите во вкладку **Actions** — если запуски идут с зелёными галочками, но сообщений нет, значит просто нет новых входящих транзакций. Если запуски не идут вообще — проверьте, не отключил ли GitHub Actions для репозитория (Settings → Actions → General).

## Технологии

- **Python 3.12** — основной скрипт
- **[mempool.space API](https://mempool.space/docs/api)** — данные о транзакциях, бесплатно, без ключа
- **[CoinGecko API](https://www.coingecko.com/en/api)** — курс BTC в USD и RUB, бесплатно, без ключа
- **[python-telegram-bot](https://python-telegram-bot.org/)** — отправка сообщений в Telegram
- **GitHub Actions** — бесплатный хостинг и планировщик задач
